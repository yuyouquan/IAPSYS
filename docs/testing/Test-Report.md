## 三、缺陷清单

### 3.1 P1 缺陷（重要）

| 缺陷编号 | 所属模块 | 缺陷描述 | 严重程度 | 复现步骤 | 建议修复方案 |
|----------|----------|----------|----------|----------|-------------|
| BUG-001 | Ant Design 兼容性 | `Steps.items.description` 在 Ant Design v6 中已废弃，应使用 `items.content` | P1 | 进入 APK 详情页，查看控制台警告 | ProcessSteps 组件中将 `description` 改为 `content` |
| BUG-002 | Ant Design 兼容性 | `Modal.destroyOnClose` 在 v6 中已废弃，应使用 `destroyOnHidden` | P1 | 打开任意节点 Modal，查看控制台警告 | 全局搜索 `destroyOnClose` 替换为 `destroyOnHidden` |
| BUG-003 | Ant Design 兼容性 | `List` 组件在 v6 中已标记废弃，将在下个大版本移除 | P1 | 打开看板状态视角，查看控制台警告 | 逐步将 `List` 替换为自定义列表或 Ant Design 推荐的替代组件 |

### 3.2 P2 缺陷（一般）

| 缺陷编号 | 所属模块 | 缺陷描述 | 严重程度 | 建议修复方案 |
|----------|----------|----------|----------|-------------|
| BUG-004 | 工作台-待办 | 待办 Alert 组件使用 `message` 属性已废弃，应改用 `title` | P2 | TodoCard 中 `Alert message` 改为 `Alert title` |
| BUG-005 | 导航菜单 | 在非工作台/看板一级路由下（如 FlowDetail、AppFlowDetail），点击顶部导航菜单"看板"不会立即跳转 | P2 | 检查 MainLayout 菜单 onClick 逻辑是否正确处理了嵌套路由 |
| BUG-006 | 面包屑 | 面包屑"工作台"链接 href="" 导致点击后 URL 变为当前页面而非 /workbench | P2 | 将 `href: ''` 改为正确的跳转路径或仅使用 onClick |
| BUG-007 | 班车名称 | 班车名称包含时间戳后缀（如"3月班车 20260301120000"），不够用户友好 | P2 | Mock 数据中班车名称去掉时间戳后缀，或在 UI 中截取显示 |
| BUG-008 | 应用图标 | 所有应用使用 📦 emoji 作为占位图标，应使用更专业的默认图标 | P2 | 替换为 Ant Design 的 AppstoreOutlined 或自定义 SVG 默认图标 |

### 3.3 P3 优化建议

| 编号 | 所属模块 | 建议内容 |
|------|----------|----------|
| OPT-001 | 构建优化 | 生产构建有 chunk > 500KB 警告，建议配置 `manualChunks` 拆分 antd 和其他大库 |
| OPT-002 | 响应式 | 待办面板在窄屏下的交互需要进一步优化，当前固定 300px 宽度 |
| OPT-003 | 加载状态 | 节点 Modal 打开时如果需要加载数据，应有 Skeleton 骨架屏 |
| OPT-004 | 键盘操作 | 流程节点条不支持键盘 Tab 导航和 Enter 打开，影响无障碍访问 |

## 六、测试结论

### 6.1 总结

IAPSYS 前端系统 v1.0 核心功能测试**通过**。系统实现了完整的工作流管理能力：

1. **工作台**: 班车列表 CRUD + 筛选搜索 + 待办面板全流程可用
2. **流程单详情**: 应用卡片展示、添加应用、状态筛选跳转正常
3. **APK 发布详情**: 7节点流程条 + 节点 Modal 交互正常
4. **看板**: 三视角数据展示正确
5. **Mock 数据**: MSW 拦截 API 请求正常，数据联通完整

### 6.2 发布建议

| 建议 | 优先级 |
|------|--------|
| 修复 3 个 Ant Design v6 废弃 API 警告 (BUG-001~003) | 发布前必须修复 |

| 修复面包屑跳转问题 (BUG-006) | 发布前修复 |
| 优化班车名称显示 (BUG-007) | 可延后 |
